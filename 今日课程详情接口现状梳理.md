# 今日课程详情接口现状梳理

## 1. 接口基础信息

### 1.1 Controller 中的 API 路径

- 主接口（今日课程详情）：`GET /courses/{campId}/today`
- 所在控制器：`AppCourseController#getTodayCourse`

### 1.2 HTTP 请求方法

- `GET`

### 1.3 接收参数

- `campId`：路径参数，类型 `Integer`，通过 `@PathVariable Integer campId` 接收。
- `userId`：不由前端显式传参，通过 `HttpServletRequest` 的 `request.getAttribute("userId")` 获取。
  - `userId` 来源于鉴权拦截器解析 JWT 后写入请求上下文。

---

## 2. 当前返回的 DTO 结构

当前接口返回 `Result<TodayCourseDTO>`，其中 `TodayCourseDTO` 包含一个 `tasks` 列表，元素类型为 `TaskItemDTO`。

### 2.1 TodayCourseDTO 全字段

1. `Boolean hasCourse`
2. `String currentDate`
3. `Integer planId`
4. `Integer completionRate`
5. `List<TaskItemDTO> tasks`

### 2.2 TaskItemDTO 全字段

1. `String taskId`
2. `String taskType`
3. `String title`
4. `String subtitle`
5. `Integer isDone`

---

## 3. 当前 SQL 实现（排课详情 + 用户完成状态）

## 3.1 排课详情 SQL（Mapper XML）

文件：`src/main/resources/mapper/CampPlanMapper.xml`  
方法：`selectTodayPlan`

```sql
SELECT 
    plan_id,
    camp_id,
    day_index,
    plan_date,
    title,
    video_url,
    graphic_url,
    module_index,
    module_name,
    reading_title,
    teacher_name,
    video_duration,
    extra_task1_name,
    extra_task2_name,
    extra_task3_name
FROM t_camp_plan
WHERE camp_id = #{campId}
  AND DATE(plan_date) = DATE(#{planDate})
```

说明：
- 该 SQL 已包含你提到的旧字段来源：`reading_title`, `video_url`, `extra_task1_name`, `extra_task2_name`, `extra_task3_name`。
- Service 层会将这些字段转换成 `TaskItemDTO` 列表后返回。

## 3.2 用户完成状态 SQL

文件：`src/main/resources/mapper/UserDailyRecordMapper.xml` 当前只有 `resultMap`，没有查询 SQL。  
实际执行查询的是 `UserDailyRecordMapper.java` 中注解 SQL：

```sql
SELECT * 
FROM t_user_daily_record 
WHERE user_id = #{userId} 
  AND plan_id = #{planId}
```

说明：
- 今日课程接口会先查到 `todayPlan.planId`，再用上述 SQL 查询该用户这一天的完成状态（`is_read_done`, `is_video_done`, `is_homework_done`, `is_extra1_done`, `is_extra2_done`, `completion_rate`）。
