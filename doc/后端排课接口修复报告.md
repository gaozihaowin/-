# 后端排课接口修复报告

## 修复概述

本次修复堵住了 **4 个数据漏水点**，覆盖新增接口（INSERT）、更新接口（UPDATE）和历史脏数据查询展示的全链路。

---

## 修复点清单

### 修复一：CampPlanSaveDayDTO（补字段）

**文件**：`src/main/java/com/daily/dailychineseculture/dto/CampPlanSaveDayDTO.java`

**变更**：在 DTO 中补全了三个之前完全缺失的字段。

```java
@NotNull(message = "模块索引不能为空")
@Schema(description = "模块索引（第几周）", example = "1")
private Integer moduleIndex;

@NotBlank(message = "模块名称不能为空")
@Schema(description = "模块名称", example = "知行合一")
private String moduleName;

@Schema(description = "讲师姓名", example = "王阳明")
private String teacherName;
```

**效果**：PUT /save-day 接口现在能接收并向下游传递周次、主题、老师三个字段。

---

### 修复二：insertCampPlan SQL（INSERT 补列）

**文件**：`src/main/resources/mapper/CampPlanMapper.xml`

**变更前**（漏了 3 个字段）：

```xml
INSERT INTO t_camp_plan (camp_id, day_index, plan_date, title)
VALUES (#{campId}, #{dayIndex}, #{planDate}, #{title})
```

**变更后**（完整 7 列）：

```xml
<insert id="insertCampPlan" parameterType="com.daily.dailychineseculture.entity.CampPlan"
        useGeneratedKeys="true" keyProperty="planId">
    INSERT INTO t_camp_plan
        (camp_id, day_index, plan_date, title, module_index, module_name, teacher_name)
    VALUES
        (#{campId}, #{dayIndex}, #{planDate}, #{title},
         #{moduleIndex}, #{moduleName}, #{teacherName})
</insert>
```

**效果**：POST /api/admin/camp-plans 新增排课时，module_index、module_name、teacher_name 正式写入数据库，不再落库为 0/NULL。

---

### 修复三：updateCampPlan SQL（UPDATE 补字段）

**文件**：`src/main/resources/mapper/CampPlanMapper.xml`

**变更前**（只更新 title）：

```xml
<update id="updateCampPlan" parameterType="com.daily.dailychineseculture.dto.CampPlanDTO">
    UPDATE t_camp_plan
    <set>
        <if test="title != null">title = #{title},</if>
    </set>
    WHERE plan_id = #{planId}
</update>
```

**变更后**（完整 7 字段动态更新）：

```xml
<update id="updateCampPlan" parameterType="com.daily.dailychineseculture.dto.CampPlanDTO">
    UPDATE t_camp_plan
    <set>
        <if test="campId != null">camp_id = #{campId},</if>
        <if test="dayIndex != null">day_index = #{dayIndex},</if>
        <if test="planDate != null">plan_date = #{planDate},</if>
        <if test="title != null and title != ''">title = #{title},</if>
        <if test="moduleIndex != null">module_index = #{moduleIndex},</if>
        <if test="moduleName != null and moduleName != ''">module_name = #{moduleName},</if>
        <if test="teacherName != null and teacherName != ''">teacher_name = #{teacherName},</if>
    </set>
    WHERE plan_id = #{planId}
</update>
```

**效果**：PUT /api/admin/camp-plans/save-day 更新排课时，所有字段均可正确回写。

### 修复五：getCourseSchedule 查询容错（防重复 + 防 null）

**文件**：`src/main/java/com/daily/dailychineseculture/service/impl/CourseServiceImpl.java`

**buildWeekName 方法变更**：

```java
private String buildWeekName(Integer moduleIndex, String moduleName) {
    // 容错 1：moduleIndex 为 null 或 <=0 时兜底为 1
    if (moduleIndex == null || moduleIndex <= 0) {
        moduleIndex = 1;
    }
    // 容错 2：moduleName 为 null 时赋予默认值
    if (moduleName == null) {
        moduleName = "拓展排课";
    }
    // 容错 3：moduleName 已含"周："说明数据库已存完整字符串，跳过拼接
    if (moduleName.contains("周：") || moduleName.contains("周:")) {
        return moduleName;
    }
    // 正常拼接逻辑
    String[] chineseNumbers = {"", "一", "二", "三", "四", "五", "六", "七", "八", "九", "十"};
    String chineseNumber = (moduleIndex >= 1 && moduleIndex <= 10)
        ? chineseNumbers[moduleIndex]
        : moduleIndex.toString();
    return "第" + chineseNumber + "周：" + moduleName;
}
```

**效果**：

| 脏数据场景 | 修复前 | 修复后 |
|-----------|--------|--------|
| `module_index=0, module_name=NULL` | `第0周: null` | `第一周：拓展排课` |
| `module_name="第一周：基础认知"`（已含前缀） | `第一周：第一周：基础认知` | `第一周：基础认知`（直接返回原值） |
