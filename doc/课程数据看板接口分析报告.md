## 3. Service 层逻辑

### 核心方法
`CourseServiceImpl.getCourseData(Integer campId, Long currentUserId)`

### 3.1 基础指标统计

#### 步骤 1：查询总天数
```java
List<CampPlan> allPlans = campPlanMapper.selectCourseScheduleByCampId(campId);
int totalDays = (allPlans != null) ? allPlans.size() : 0;
dto.setTotalDays(totalDays);
```

#### 步骤 2：查询用户打卡记录
```java
List<UserDailyRecord> recordList = userDailyRecordMapper.selectByUserIdAndCampId(currentUserId, campId);
if (recordList == null) {
    recordList = new ArrayList<>();
}
```

#### 步骤 3：统计已完成天数
```java
int completedDays = 0;
for (UserDailyRecord record : recordList) {
    if (record.getCompletionRate() != null && record.getCompletionRate() == 100) {
        completedDays++;
    }
}
dto.setCompletedDays(completedDays);
```

**计算逻辑**：遍历用户在该营期的所有打卡记录，统计 `completionRate == 100` 的记录数。

#### 步骤 4：计算总体完成率
```java
long overallRate = (totalDays > 0) ? Math.round((completedDays * 100.0) / totalDays) : 0L;
dto.setOverallRate((int) overallRate);
```

**计算公式**：
```
overallRate = (completedDays / totalDays) × 100
```

### 3.2 学习趋势推导（trends 列表组装）

#### 核心逻辑
```java
List<TrendItemDTO> trends = new ArrayList<>();
java.time.LocalDate today = java.time.LocalDate.now();

// 查询营期全部课次
List<CampPlan> allTrendPlans = campPlanMapper.selectRecentPlansByCampId(campId, java.sql.Date.valueOf(today));

if (allTrendPlans != null && !allTrendPlans.isEmpty()) {
    for (CampPlan plan : allTrendPlans) {
        TrendItemDTO trendItem = new TrendItemDTO();
        trendItem.setDayStr("Day" + plan.getDayIndex());
        trendItem.setDayIndex(plan.getDayIndex());

        // 查找该计划的完成率
        Integer rate = 0;
        for (UserDailyRecord record : recordList) {
            if (record.getPlanId().equals(plan.getPlanId())) {
                rate = (record.getCompletionRate() != null) ? record.getCompletionRate() : 0;
                break;
            }
        }
        trendItem.setRate(rate);

        // 判断日期
        java.time.LocalDate planDate = (plan.getPlanDate() == null)
                ? today
                : plan.getPlanDate().toInstant().atZone(java.time.ZoneId.systemDefault()).toLocalDate();

        // 状态判断逻辑
        if (planDate.isAfter(today)) {
            trendItem.setStatus("LOCKED");   // 未解锁
            trendItem.setRate(0);
        } else if (rate > 0) {
            trendItem.setStatus("COMPLETED"); // 已完成
        } else {
            trendItem.setStatus("MISSED");    // 漏打卡
        }

        trends.add(trendItem);
    }
}
dto.setTrends(trends);
```

#### 状态判断规则

| 条件 | 状态 | 说明 |
|------|------|------|
| `planDate > today` | `LOCKED` | 未来日期，未解锁 |
| `rate > 0` | `COMPLETED` | 已有打卡记录（完成率 > 0） |
| `rate == 0` | `MISSED` | 过去日期但未打卡 |

> **注意**：当前逻辑中，只要 `rate > 0` 就标记为 `COMPLETED`，即使 `rate < 100`（部分完成）也会显示为已完成状态。

### 3.3 成就徽章计算

```java
List<AchievementDTO> achievements = new ArrayList<>();

// 规则 1: 完成 >= 1 天
if (completedDays >= 1) {
    AchievementDTO beginner = new AchievementDTO();
    beginner.setIcon("https://img.icons8.com/color/96/medal2.png");
    beginner.setTitle("初学者");
    beginner.setDesc("完成第一天学习，开启致良知之旅");
    achievements.add(beginner);
}

// 规则 2: 完成 >= 3 天
if (completedDays >= 3) {
    AchievementDTO progress = new AchievementDTO();
    progress.setIcon("https://img.icons8.com/color/96/warranty.png");
    progress.setTitle("渐入佳境");
    progress.setDesc("累计完成三天学习");
    achievements.add(progress);
}

// 规则 3: 100% 完成（圆满结业）
if (totalDays > 0 && completedDays == totalDays) {
    AchievementDTO trophy = new AchievementDTO();
    trophy.setIcon("https://img.icons8.com/color/96/trophy.png");
    trophy.setTitle("圆满结业");
    trophy.setDesc("完成全部课程");
    achievements.add(trophy);
}

dto.setAchievements(achievements);
```

## 4. SQL 实现

### 4.1 查询营期所有排课计划（CampPlanMapper.xml）

```xml
<select id="selectCourseScheduleByCampId" resultType="com.daily.dailychineseculture.entity.CampPlan">
    SELECT 
        plan_id,
        camp_id,
        day_index,
        plan_date,
        title,
        module_index,
        module_name,
        teacher_name,
        is_finished
    FROM t_camp_plan
    WHERE camp_id = #{campId}
    ORDER BY day_index ASC
</select>
```

**用途**：获取营期总天数（排课总数）。

### 4.3 查询用户历史打卡记录（UserDailyRecordMapper.java）

```java
@Select("SELECT record_id, user_id, camp_id, plan_id, completion_rate, is_all_completed " +
        "FROM t_user_daily_record " +
        "WHERE user_id = #{userId} AND camp_id = #{campId}")
List<UserDailyRecord> selectByUserIdAndCampId(@Param("userId") Long userId, @Param("campId") Integer campId);
```

**用途**：获取用户在该营期的所有打卡汇总记录，用于：
1. 统计 `completedDays`
2. 为每个 `trendItem` 匹配对应的 `rate`

## 7. 潜在问题与优化建议

### 问题 1：trends 状态判断不够精确
当前逻辑：`rate > 0` → `COMPLETED`

**问题**：如果用户只完成了 50% 的任务，也会被标记为 `COMPLETED`，这可能不是预期行为。

**建议**：区分 `IN_PROGRESS`（进行中）和 `COMPLETED`（已完成）状态：
```java
if (rate == 100) {
    trendItem.setStatus("COMPLETED");
} else if (rate > 0) {
    trendItem.setStatus("IN_PROGRESS");
} else {
    trendItem.setStatus("MISSED");
}
```

### 问题 2：性能问题
当前逻辑在循环中遍历 `recordList` 查找匹配的 `planId`，时间复杂度为 O(n²)。

**建议**：预先将 `recordList` 转换为 `Map<planId, UserDailyRecord>`，将查找复杂度降为 O(1)。

### 问题 3：selectRecentPlansByCampId 参数未使用
方法签名中有 `planDate` 参数，但 SQL 中未使用该参数进行过滤，实际返回的是营期全部课次。

**建议**：如果确实需要限制返回数量，应在 SQL 中添加条件过滤。
