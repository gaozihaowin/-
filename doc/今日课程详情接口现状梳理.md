**方法签名**：

### 1.4 SQL 实现

#### 查询今日排课（CampPlanMapper.xml）

```xml
<select id="selectTodayPlan" resultType="com.daily.dailychineseculture.entity.CampPlan">
    SELECT 
        plan_id,
        camp_id,
        day_index,
        plan_date,
        title,
        module_index,
        module_name,
        teacher_name,
        is_finished
    FROM t_camp_plan
    WHERE camp_id = #{campId}
      AND DATE(plan_date) = DATE(#{planDate})
</select>
```

#### 查询任务列表及完成状态（PlanTaskMapper.xml）

```xml
<select id="selectTaskItemsByPlanIdAndUserId" resultType="com.daily.dailychineseculture.dto.TaskItemDTO">
    SELECT
        t.task_id AS taskId,
        t.task_type AS taskType,
        t.task_name AS title,
        t.task_desc AS subtitle,
        t.is_required AS isRequired,
        IF(r.record_id IS NOT NULL, 1, 0) AS isDone
    FROM t_plan_task t
    LEFT JOIN t_user_task_record r ON t.task_id = r.task_id
        AND r.user_id = #{userId}
        AND r.is_done = 1
    WHERE t.plan_id = #{planId}
      AND t.is_deleted = 0
    ORDER BY t.sort_order ASC
</select>
```

## 2. 升维改造方案评估

### 2.1 需求背景

前端"今日课程"面板即将升级为"通用单日任务看板"，需要支持：
- 查询"今天"的课程（现有功能）
- 查询"历史任意一天"的课程（新增需求）

### 2.2 方案对比

#### 方案 A：扩展现有接口

```java
@GetMapping("/{campId}/today")
public Result<TodayCourseDTO> getTodayCourse(
    @PathVariable Integer campId,
    @RequestParam(required = false) Integer planId,  // 新增可选参数
    HttpServletRequest request
)
```

**逻辑**：
- 如果 `planId != null`：直接根据 `planId` 查询指定课次
- 如果 `planId == null`：保持原有逻辑，查询今天的课次

**优点**：
- 接口路径不变，前端改动最小
- 复用现有 DTO 结构，无需新增
- 代码改动量小，只需修改 Service 层逻辑

**缺点**：
- 接口语义不够清晰（`/today` 却可以查历史）
- 违反 RESTful 设计原则（URI 应该表达资源语义）

---

#### 方案 B：新增独立接口

```java
@GetMapping("/plan/{planId}/detail")
public Result<TodayCourseDTO> getPlanDetail(
    @PathVariable Integer planId,
    HttpServletRequest request
)
```

**逻辑**：
- 直接根据 `planId` 查询指定课次的详情
- 原有 `/today` 接口保持不变

**优点**：
- 接口语义清晰，符合 RESTful 设计
- 职责分离，便于维护
- 原有接口不受影响，降低回归风险

**缺点**：
- 需要新增接口，前端需要适配
- Service 层可能存在代码重复（可通过抽取公共方法解决）

---

### 2.3 专业建议

**推荐方案 A（扩展现有接口）**

**理由**：

1. **前端改动最小**：只需在调用时增加可选参数 `planId`，无需修改接口路径
2. **DTO 完美复用**：`TodayCourseDTO` 的字段结构完全适用于"单日任务看板"，无需新增 DTO
3. **代码改动量小**：只需修改 Service 层逻辑，Controller 层增加一个参数即可
4. **向后兼容**：不传 `planId` 时保持原有行为，不影响现有功能

**实现建议**：

```java
@Override
public TodayCourseDTO getTodayCourse(Integer campId, Long currentUserId, Integer planId) {
    java.time.LocalDate today = java.time.LocalDate.now();
    String currentDate = today.format(java.time.format.DateTimeFormatter.ofPattern("M 月 d 日"));
    
    CampPlan targetPlan;
    
    if (planId != null) {
        // 指定 planId：直接查询
        targetPlan = campPlanMapper.selectById(planId);
        if (targetPlan == null) {
            throw new IllegalArgumentException("排课计划不存在，planId: " + planId);
        }
        // 可选：校验 planId 是否属于该 campId
        if (!targetPlan.getCampId().equals(campId)) {
            throw new IllegalArgumentException("该排课不属于指定营期");
        }
    } else {
        // 未指定 planId：查询今日排课
        targetPlan = campPlanMapper.selectTodayPlan(campId, java.sql.Date.valueOf(today));
    }
    
    // 后续逻辑与原有一致...
}
```

### 2.4 DTO 复用确认

两种方案都可以**完美复用**现有的 `TodayCourseDTO` 结构：

| 字段 | 今日课程 | 历史课程 | 是否复用 |
|------|----------|----------|----------|
| `hasCourse` | ✅ | ✅ | 是 |
| `currentDate` | 今日日期 | 历史日期 | 是 |
| `planId` | ✅ | ✅ | 是 |
| `completionRate` | ✅ | ✅ | 是 |
| `tasks` | ✅ | ✅ | 是 |

**结论**：无论选择哪种方案，响应 DTO 结构无需任何修改。

### 6.4 Service 层核心逻辑

```java
@Override
public TodayCourseDTO getTodayCourse(Integer campId, Long currentUserId, Integer planId) {
    CampPlan targetPlan = null;
    String displayDateStr = "";

    if (planId != null) {
        // 时光机模式：根据 planId 查询历史天
        targetPlan = campPlanMapper.selectById(planId);
        if (targetPlan == null || !targetPlan.getCampId().equals(campId)) {
            TodayCourseDTO emptyDto = new TodayCourseDTO();
            emptyDto.setHasCourse(false);
            emptyDto.setCurrentDate("");
            emptyDto.setPlanId(null);
            emptyDto.setCompletionRate(0);
            emptyDto.setTasks(new ArrayList<>());
            return emptyDto;
        }
        if (targetPlan.getPlanDate() != null) {
            displayDateStr = targetPlan.getPlanDate().toInstant()
                    .atZone(java.time.ZoneId.systemDefault())
                    .toLocalDate()
                    .format(java.time.format.DateTimeFormatter.ofPattern("M 月 d 日"));
        } else {
            displayDateStr = "Day " + targetPlan.getDayIndex();
        }
    } else {
        // 常规模式：查询今天
        java.time.LocalDate today = java.time.LocalDate.now();
        displayDateStr = today.format(java.time.format.DateTimeFormatter.ofPattern("M 月 d 日"));
        targetPlan = campPlanMapper.selectTodayPlan(campId, java.sql.Date.valueOf(today));
    }

    if (targetPlan == null) {
        TodayCourseDTO emptyDto = new TodayCourseDTO();
        emptyDto.setHasCourse(false);
        emptyDto.setCurrentDate(displayDateStr);
        emptyDto.setPlanId(null);
        emptyDto.setCompletionRate(0);
        emptyDto.setTasks(new ArrayList<>());
        return emptyDto;
    }

    TodayCourseDTO dto = new TodayCourseDTO();
    dto.setHasCourse(true);
    dto.setCurrentDate(displayDateStr);
    dto.setPlanId(targetPlan.getPlanId());

    List<TaskItemDTO> tasks = planTaskMapper.selectTaskItemsByPlanIdAndUserId(targetPlan.getPlanId(), currentUserId);
    if (tasks == null) {
        tasks = new ArrayList<>();
    }
    dto.setTasks(tasks);

    UserDailyRecord summary = userDailyRecordMapper.selectByUserIdAndPlanId(currentUserId, targetPlan.getPlanId());
    int completionRate = 0;
    if (summary != null && summary.getCompletionRate() != null) {
        completionRate = summary.getCompletionRate();
    } else if (!tasks.isEmpty()) {
        int doneCount = 0;
        for (TaskItemDTO task : tasks) {
            if (task.getIsDone() != null && task.getIsDone() == 1) {
                doneCount++;
            }
        }
        completionRate = doneCount * 100 / tasks.size();
    }
    dto.setCompletionRate(completionRate);
    return dto;
}
```

### 7.4 核心代码实现

```java
@Override
public TodayCourseDTO getTodayCourse(Integer campId, Long currentUserId, Integer planId) {
    
    // ═══════════════════════════════════════════════════════════════════
    // 步骤一：排课计划 (CampPlan) 路由
    // ═══════════════════════════════════════════════════════════════════
    
    CampPlan targetPlan = null;
    String displayDateStr = "";

    if (planId != null) {
        // 【时光机模式】根据 planId 查询指定排课
        targetPlan = campPlanMapper.selectById(planId);
        
        // 校验排课是否存在且属于当前营期
        if (targetPlan == null || !targetPlan.getCampId().equals(campId)) {
            TodayCourseDTO emptyDto = new TodayCourseDTO();
            emptyDto.setHasCourse(false);
            emptyDto.setCurrentDate("");
            emptyDto.setPlanId(null);
            emptyDto.setCompletionRate(0);
            emptyDto.setTasks(new ArrayList<>());
            return emptyDto;
        }
        
        // 使用排课的真实日期作为展示日期
        if (targetPlan.getPlanDate() != null) {
            displayDateStr = targetPlan.getPlanDate().toInstant()
                    .atZone(java.time.ZoneId.systemDefault())
                    .toLocalDate()
                    .format(java.time.format.DateTimeFormatter.ofPattern("M 月 d 日"));
        } else {
            // 兜底：使用 Day 索引
            displayDateStr = "Day " + targetPlan.getDayIndex();
        }
    } else {
        // 【常规模式】查询今日排课
        java.time.LocalDate today = java.time.LocalDate.now();
        displayDateStr = today.format(java.time.format.DateTimeFormatter.ofPattern("M 月 d 日"));
        targetPlan = campPlanMapper.selectTodayPlan(campId, java.sql.Date.valueOf(today));
    }

    // ═══════════════════════════════════════════════════════════════════
    // 步骤二：空状态兜底
    // ═══════════════════════════════════════════════════════════════════
    
    if (targetPlan == null) {
        TodayCourseDTO emptyDto = new TodayCourseDTO();
        emptyDto.setHasCourse(false);
        emptyDto.setCurrentDate(displayDateStr);
        emptyDto.setPlanId(null);
        emptyDto.setCompletionRate(0);
        emptyDto.setTasks(new ArrayList<>());
        return emptyDto;
    }

    // ═══════════════════════════════════════════════════════════════════
    // 步骤三：复用原有逻辑（查询任务列表 & 完成率）
    // ═══════════════════════════════════════════════════════════════════
    
    TodayCourseDTO dto = new TodayCourseDTO();
    dto.setHasCourse(true);
    dto.setCurrentDate(displayDateStr);
    dto.setPlanId(targetPlan.getPlanId());

    // 查询任务列表（完全复用原有 Mapper）
    List<TaskItemDTO> tasks = planTaskMapper.selectTaskItemsByPlanIdAndUserId(
            targetPlan.getPlanId(), 
            currentUserId
    );
    if (tasks == null) {
        tasks = new ArrayList<>();
    }
    dto.setTasks(tasks);

    // 查询完成率（完全复用原有 Mapper）
    UserDailyRecord summary = userDailyRecordMapper.selectByUserIdAndPlanId(
            currentUserId, 
            targetPlan.getPlanId()
    );
    int completionRate = 0;
    if (summary != null && summary.getCompletionRate() != null) {
        completionRate = summary.getCompletionRate();
    } else if (!tasks.isEmpty()) {
        // 兜底：根据任务完成情况计算
        int doneCount = 0;
        for (TaskItemDTO task : tasks) {
            if (task.getIsDone() != null && task.getIsDone() == 1) {
                doneCount++;
            }
        }
        completionRate = doneCount * 100 / tasks.size();
    }
    dto.setCompletionRate(completionRate);
    
    return dto;
}
```

### 7.5 设计要点总结

| 要点 | 说明 |
|------|------|
| **参数可选性** | `planId` 为可选参数，不传时保持原有行为，确保向后兼容 |
| **归属校验** | 时光机模式下校验 `planId` 是否属于当前 `campId`，防止越权访问 |
| **日期展示** | 时光机模式使用排课真实日期，常规模式使用系统当前日期 |
| **逻辑复用** | 获取 `targetPlan` 后，后续查询任务和完成率的逻辑完全不变 |
| **DTO 复用** | 完全复用 `TodayCourseDTO`，无需新增或修改任何 DTO 字段 |
| **空状态处理** | 无排课时返回 `hasCourse=false` 的空状态 DTO |
