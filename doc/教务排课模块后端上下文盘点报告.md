## 2. 现有接口逻辑盘点

### 2.1 CampPlanController 已有 API 接口一览

| 序号 | 请求方式 | 路径 | @RequestBody 结构 | 功能说明 |
|------|----------|------|-------------------|----------|
| 1 | `GET` | `/api/admin/camp-plans?campId={campId}` | 无 | 获取某营期的排课时间轴 |
| 2 | `POST` | `/api/admin/camp-plans/generate` | `GenerateCalendarRequest` | 一键生成空日历 |
| 3 | `POST` | `/api/admin/camp-plans` | `CampPlanDTO` | 新增一天的排课 |
| 4 | `PUT` | `/api/admin/camp-plans` | `CampPlanDTO` | 保存/更新单日课表 |
| 5 | `DELETE` | `/api/admin/camp-plans/{planId}` | 无 | 删除整天排课及挂载的所有任务 |
| 6 | `PUT` | `/api/admin/camp-plans/save-day` | `CampPlanSaveDayDTO` | 聚合保存单日排课（主表+任务列表全量刷新） |
| 7 | `POST` | `/api/admin/camp-plans/append` | `AppendCampPlanRequest` | 追加一天排课（系统自动推算 dayIndex 和 planDate） |
| **8** | **`POST`** | **`/api/admin/camp-plans/add-smart-day`** | **`CampPlanAddDayDTO`** | **【新增】智能追加排课（前端推算完整数据后落库）** |

### 2.2 新增接口详情：智能追加排课

#### 接口 #8：智能追加一天排课（新增）

```http
POST /api/admin/camp-plans/add-smart-day
Content-Type: application/json
@Validated

{
  "campId": 101,
  "dayIndex": 8,
  "planDate": "2026-03-08",
  "moduleName": "第二周：知行合一",
  "moduleIndex": 2,
  "teacherName": "博仁老师",
  "title": "第8课：知行本体"
}
```

**Service 方法**：`CampPlanService.addSmartDay(CampPlanAddDayDTO requestDTO)`

**核心逻辑**：
1. 校验营期是否存在
2. 使用 `BeanUtils.copyProperties` 将 DTO 属性拷贝到实体
3. 强制设置 `isFinished = 0`（默认未完成）
4. **铁桶防御**：捕获 `DuplicateKeyException`，返回友好错误提示

**成功响应**：
```json
{
  "code": 200,
  "message": "智能追加排课成功",
  "data": null
}
```

**并发冲突响应（409）**：
```json
{
  "code": 409,
  "message": "该营期的当前天数或日期已被占用，请刷新页面获取最新排课进度！"
}
```
