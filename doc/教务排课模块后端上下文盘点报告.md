## 1. 数据库与实体类盘点

### 1.1 CampPlan 实体类（对应数据库表 `t_camp_plan`）

| 字段名 | Java 类型 | 说明 | 数据库 NULL 情况 |
|--------|-----------|------|-----------------|
| `planId` | `Integer` | 计划 ID（主键） | NOT NULL |
| `campId` | `Integer` | 营期 ID | NOT NULL |
| `dayIndex` | `Integer` | 第几天 | **需确认** |
| `planDate` | `java.util.Date` | 具体日期 | **需确认** |
| `title` | `String` | 导读主题 | **需确认** |
| `moduleIndex` | `Integer` | 模块索引（第几周） | **需确认** |
| `moduleName` | `String` | 模块名称 | **需确认** |
| `teacherName` | `String` | 讲师姓名 | **需确认** |
| `isFinished` | `Integer` | 是否完成：0-未完成，1-已完成 | **需确认** |

> ⚠️ **注意**：上述 `**需确认**` 的字段，建议通过查看数据库表结构或 MyBatis Mapper XML 文件确认是否允许 NULL。

### 1.2 CampPlanDTO（前端交互用 DTO）

| 字段名 | Java 类型 | 说明 |
|--------|-----------|------|
| `planId` | `Integer` | 计划 ID |
| `campId` | `Integer` | 营期 ID |
| `dayIndex` | `Integer` | 第几天 |
| `planDate` | `java.util.Date` | 具体日期 |
| `title` | `String` | 导读标题 |
| `tasks` | `List<PlanTaskDTO>` | 任务列表（该排课下的所有任务：视频、阅读、作业等） |

> ⚠️ **重要差异**：`CampPlanDTO` 相比 `CampPlan` 实体类，**缺少了以下字段**：
> - `moduleIndex`
> - `moduleName`
> - `teacherName`
> - `isFinished`
>
> 如果新增的 `add-day` 接口需要传递这些字段，需要新建 DTO 或扩展现有 DTO。

## 2. 现有接口逻辑盘点

### 2.1 CampPlanController 已有 API 接口一览

| 序号 | 请求方式 | 路径 | @RequestBody 结构 | 功能说明 |
|------|----------|------|-------------------|----------|
| 1 | `GET` | `/api/admin/camp-plans?campId={campId}` | 无 | 获取某营期的排课时间轴 |
| 2 | `POST` | `/api/admin/camp-plans/generate` | `GenerateCalendarRequest` | 一键生成空日历 |
| 3 | `POST` | `/api/admin/camp-plans` | `CampPlanDTO` | 新增一天的排课 |
| 4 | `PUT` | `/api/admin/camp-plans` | `CampPlanDTO` | 保存/更新单日课表 |
| 5 | `DELETE` | `/api/admin/camp-plans/{planId}` | 无 | 删除整天排课及挂载的所有任务 |
| 6 | `PUT` | `/api/admin/camp-plans/save-day` | `CampPlanSaveDayDTO` | 聚合保存单日排课（主表+任务列表全量刷新） |
| 7 | `POST` | `/api/admin/camp-plans/append` | `AppendCampPlanRequest` | 追加一天排课（系统自动推算 dayIndex 和 planDate） |

### 2.2 现有新增/更新相关接口详情

#### 接口 #3：新增一天的排课
```http
POST /api/admin/camp-plans
Content-Type: application/json

{
  "campId": 101,
  "dayIndex": 1,
  "planDate": "2026-04-01",
  "title": "开营第一天"
}
```
- Service 方法：`CampPlanService.addCampPlan(CampPlanDTO campPlan)`
- 返回：`ResponseResult<CampPlanDTO>`（包含新增后的 planId）

#### 接口 #6：聚合保存单日排课
```http
PUT /api/admin/camp-plans/save-day
Content-Type: application/json

{
  "id": 10211,
  "campId": 101,
  "title": "第一天课程",
  "tasks": [
    {
      "taskId": null,
      "taskName": "视频学习",
      "taskType": "VIDEO",
      "taskDesc": "观看视频",
      "duration": 30,
      "isRequired": 1
    }
  ]
}
```
- Service 方法：`CampPlanService.saveDayPlan(CampPlanSaveDayDTO request)`

## 3. 业务层扩展性评估

### 3.1 Mapper 层可复用方法

| 方法名 | 说明 | 是否可复用 |
|--------|------|-----------|
| `insertCampPlan(CampPlan campPlan)` | 插入单条排课计划实体 | ✅ **可直接复用** |
| `batchInsertCampPlans(List<CampPlan> campPlans)` | 批量插入排课计划 | ✅ 可用于批量场景 |
| `updateCampPlan(CampPlanDTO campPlan)` | 根据 ID 更新排课计划 | ⚠️ 参数是 DTO，需适配 |
| `selectById(Integer planId)` | 根据 ID 查询排课计划 | ✅ 可用于数据回显 |
| `selectMaxDayIndexByCampId(Integer campId)` | 查询某营期最大 day_index | ✅ 可用于推算 |
| `selectMaxPlanDateByCampId(Integer campId)` | 查询某营期最晚 plan_date | ✅ 可用于推算 |

### 3.2 Service 层可复用方法

| 方法名 | 说明 | 是否可复用 |
|--------|------|-----------|
| `addCampPlan(CampPlanDTO campPlan)` | 新增一天的排课 | ✅ **可直接复用** |
| `saveOrUpdateCampPlan(CampPlanDTO campPlan)` | 保存/更新单日课表 | ⚠️ 适合更新场景 |
| `saveDayPlan(CampPlanSaveDayDTO request)` | 聚合保存单日排课 | ⚠️ 包含任务同步逻辑 |
| `appendDay(Integer campId)` | 追加一天（自动推算） | ⚠️ 自动推算逻辑已封装 |

### 3.3 关键结论

1. **插入单条记录**：可以直接使用 `CampPlanMapper.insertCampPlan(CampPlan)` 方法。
2. **CampPlanDTO 缺少字段**：如果要传递 `moduleIndex`、`moduleName`、`teacherName`，需要：
   - 方案 A：扩展 `CampPlanDTO` 添加这些字段
   - 方案 B：新建 `CampPlanAddDayDTO` 专门用于 `add-day` 接口
3. **新增接口建议**：建议新建 `AddDayRequest` DTO，包含前端智能推算的完整数据。
