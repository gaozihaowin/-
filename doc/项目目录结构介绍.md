# 项目目录结构介绍

## 一、项目概述

**项目名称**: Daily Chinese Culture (每日中国文化)  
**技术栈**: Spring Boot 4.0.2 + MyBatis + MySQL + Java 21  
**项目类型**: 在线教育教务管理系统后端服务  
**运行端口**: 8080

## 三、核心模块说明

### 1. **Controller 层（控制器）** - 25 个接口类

负责处理 HTTP 请求，调用 Service 层处理业务逻辑，返回统一格式的响应结果。

**核心接口**:
- `CampPlanController` - 教务排课工作台（新增 ⭐）
- `CampTypeController` - 课程体系分类管理（新增 ⭐）
- `CampController` - 营期管理
- `UserController` - 用户管理
- `CourseController` - 课程管理
- `AuthController` - 认证授权
- `AdminController` - 后台管理

### 2. **Service 层（业务逻辑）** - 19 个服务接口 + 17 个实现类

封装核心业务逻辑，处理数据校验、事务控制、业务规则等。

**核心服务**:
- `CampPlanService` - 排课计划服务（新增 ⭐）
- `CampTypeService` - 课程体系分类服务（新增 ⭐）
- `UserService` - 用户服务
- `CampService` - 营期服务
- `IdGeneratorService` - ID 生成器服务

### 3. **Mapper 层（数据访问）** - 21 个数据访问接口

负责与数据库交互，执行 CRUD 操作。

**核心 Mapper**:
- `CampPlanMapper` - 排课计划数据访问（新增 ⭐）
- `CampTypeMapper` - 课程体系分类数据访问（新增 ⭐）
- `CampMapper` - 营期数据访问（新增 INSERT/UPDATE ⭐）
- `UserMapper` - 用户数据访问

### 4. **Entity 层（实体类）** - 12 个实体类

对应数据库表的 Java 对象表示。

**核心实体**:
- `CampPlan` - 排课计划实体（新增 ⭐）
- `CampType` - 课程体系分类实体（新增 ⭐）
- `Camp` - 营期实体（新增 term、tag 字段 ⭐）
- `User` - 用户实体

### 5. **DTO 层（数据传输对象）** - 66 个 DTO 类

用于各层之间的数据传输，避免直接暴露实体类。

**核心 DTO** (新增 ⭐):
- `CampPlanDTO` - 排课计划 DTO
- `CampOptionDTO` - 营期下拉选项 DTO（新增 term 字段 ⭐）
- `GenerateCalendarRequest` - 生成日历请求 DTO
- `CampTypeDTO` - 课程体系分类 DTO
- `CampDTO` - 营期管理 DTO（新增 ⭐，含日期格式注解）
- `CampListItemDTO` - 营期列表项 DTO（新增 typeId 字段 ⭐）

### 6. **Common 层（公共组件）** - 3 个公共类

提供全局异常处理、统一响应封装等公共功能。

**核心组件**:
- `ResponseResult` - 统一响应结果封装（含 code、msg、data）
- `GlobalExceptionHandler` - 全局异常处理器
- `Result` - 简化版响应结果

### 7. **Config 层（配置类）** - 1 个配置类

提供 Spring Boot 应用的各种配置。

**核心配置**:
- `WebConfig` - Web 配置（CORS 跨域、拦截器注册等）

### 8. **Interceptor 层（拦截器）** - 2 个拦截器

提供请求拦截和权限校验功能。

**核心拦截器**:
- `AuthInterceptor` - 通用认证拦截器（JWT  Token 校验）
- `AdminAuthInterceptor` - 管理员权限拦截器

### 9. **Util 层（工具类）** - 1 个工具类

提供通用的工具方法。

**核心工具**:
- `JwtUtils` - JWT Token 生成和解析工具

## 四、新增功能模块（⭐ 标记）

### 1. **课程体系分类管理模块**
- **位置**: `controller/CampTypeController.java`
- **功能**: 提供 RESTful CRUD 接口
  - `GET /api/admin/camp-types` - 查询全量列表
  - `POST /api/admin/camp-types` - 新增分类
  - `PUT /api/admin/camp-types` - 修改分类
  - `DELETE /api/admin/camp-types/{typeId}` - 删除分类
- **相关文件**:
  - Entity: `CampType.java` (新增 level 字段)
  - DTO: `CampTypeDTO.java`
  - Mapper: `CampTypeMapper.java` + `CampTypeMapper.xml`
  - Service: `CampTypeService.java` + `CampTypeServiceImpl.java`

### 2. **教务排课工作台模块**
- **位置**: `controller/CampPlanController.java`
- **功能**: 提供排课相关接口
  - `GET /api/admin/camps/options` - 获取营期下拉选项
  - `GET /api/admin/camp-plans?campId={campId}` - 获取排课时间轴
  - `POST /api/admin/camp-plans/generate` - 一键生成空日历
  - `PUT /api/admin/camp-plans` - 保存/更新单日课表
- **相关文件**:
  - Entity: `CampPlan.java`
  - DTO: `CampPlanDTO.java`, `CampOptionDTO.java`, `GenerateCalendarRequest.java`
  - Mapper: `CampPlanMapper.java` + `CampPlanMapper.xml`
  - Service: `CampPlanService.java` + `CampPlanServiceImpl.java`

### 3. **营期管理 CRUD 模块** ⭐
- **位置**: `controller/CampController.java`
- **功能**: 提供营期新增和编辑接口
  - `POST /api/admin/camps` - 新增营期（enroll_count 强制为 0）
  - `PUT /api/admin/camps` - 编辑营期（不更新 enroll_count）
  - `GET /api/admin/camps/options` - 获取营期下拉选项（含 term 期数）
  - `GET /api/admin/camps?page=1&size=10` - 分页查询营期列表（含 typeId 精准回显）
- **相关文件**:
  - Entity: `Camp.java`（新增 term、tag 字段）
  - DTO: `CampDTO.java`（新增日期格式注解）、`CampListItemDTO.java`（新增 typeId）
  - Mapper: `CampMapper.java` + `CampMapper.xml`（新增 INSERT/UPDATE）
  - Service: `CampService.java` + `CampServiceImpl.java`（新增 addCamp/updateCamp）

## 五、技术架构分层

```
┌─────────────────────────────────────┐
│     Controller 层 (REST API)        │  ← HTTP 请求处理
├─────────────────────────────────────┤
│     Service 层 (业务逻辑)           │  ← 业务规则、事务控制
├─────────────────────────────────────┤
│     Mapper 层 (数据访问)            │  ← SQL 执行、CRUD 操作
├─────────────────────────────────────┤
│     Entity 层 (实体对象)            │  ← 数据库表映射
└─────────────────────────────────────┘
         ↓
┌─────────────────────────────────────┐
│        MySQL 数据库                 │
└─────────────────────────────────────┘
```

**辅助层**:
- **DTO/VO**: 数据传输对象，解耦各层之间的依赖
- **Interceptor**: 拦截器，处理权限校验
- **Common**: 统一响应封装和异常处理
- **Util**: 工具类，提供通用功能

## 十、项目特色

1. **标准的 RESTful API 设计**: 所有接口遵循 RESTful 规范
2. **统一响应格式**: 使用 `ResponseResult` 封装统一响应
3. **完善的分层架构**: Controller → Service → Mapper → Entity
4. **丰富的 DTO 设计**: 避免直接暴露实体类，提高安全性
5. **全面的文档覆盖**: 每个功能模块都有详细的 API文档
6. **事务控制**: 关键业务操作使用 `@Transactional` 注解
7. **JWT 认证**: 使用 JWT Token 进行用户认证和权限校验
8. **MyBatis XML 映射**: SQL 与代码分离，便于维护
9. **数据保护机制**: 
   - 新增营期时强制设置 enroll_count = 0 ⭐
   - 编辑营期时不更新 enroll_count，保护真实数据 ⭐

    - 列表查询返回 typeId，支持前端下拉框精准选中 ⭐
    - CampOptionDTO 包含 term 期数，显示完整信息 ⭐
11. **日期格式处理**:
