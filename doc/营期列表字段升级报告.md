# 营期列表查询接口字段升级报告

## 升级背景

为了完善营期管理（Camp）的列表回显和数据严谨性，特别是确保前端编辑时能够精准回显体系类型，对 `GET /api/admin/camps` 接口的返回 DTO 进行了升级。

---

## 核心修改

### 1️⃣ **CampListItemDTO.java** - 添加 typeId 字段

```java
package com.daily.dailychineseculture.dto;

import com.fasterxml.jackson.annotation.JsonProperty;
import lombok.Data;

/**
 * 营期列表项 DTO
 * 用于 PC 端后台管理系统 - 营期管理大盘
 */
@Data
public class CampListItemDTO {
    
    /**
     * 营期 ID
     */
    @JsonProperty("campId")
    private Integer campId;
    
    /**
     * 体系 ID（来自 t_camp_type.type_id）
     * 【重要】用于前端编辑时的下拉框精准回显
     */
    @JsonProperty("typeId")
    private Integer typeId;  // ← 新增字段
    
    /**
     * 类型名称（来自 t_camp_type.level_name）
     */
    @JsonProperty("typeName")
    private String typeName;
    
    /**
     * 期数
     */
    @JsonProperty("term")
    private Integer term;
    
    /**
     * 营期名称
     */
    @JsonProperty("name")
    private String name;
    
    /**
     * 开营时间（格式：yyyy-MM-dd）
     */
    @JsonProperty("startTime")
    private String startTime;
    
    /**
     * 结营时间（格式：yyyy-MM-dd）
     */
    @JsonProperty("endTime")
    private String endTime;
    
    /**
     * 状态：0-待开课，1-进行中，2-已结束
     */
    @JsonProperty("status")
    private Integer status;
    
    /**
     * 标签（如：热招）
     */
    @JsonProperty("tag")
    private String tag;
    
    /**
     * 报名人数
     */
    @JsonProperty("enrollCount")
    private Integer enrollCount;
}
```

**关键变更**:
- ✅ 新增 `typeId` 字段（Integer 类型）
- ✅ 使用 `@JsonProperty("typeId")` 确保 JSON 输出为驼峰命名
- ✅ 添加注释强调其重要性：**用于前端编辑时的下拉框精准回显**

```xml
    <result property="endTime" column="end_time"/>
    <result property="status" column="status"/>
    <result property="tag" column="tag"/>

```

#### **SQL 查询语句升级**
```xml
<!-- 分页查询营期列表（联表查询） -->
<select id="selectCampList" resultMap="CampListItemResultMap">
    SELECT 
        c.camp_id,
        c.type_id,              <!-- ← 新增：查询营期的体系 ID -->
        ct.level_name AS level_name,
        c.term,
        c.name,
        DATE_FORMAT(c.start_time, '%Y-%m-%d') AS start_time,
        DATE_FORMAT(c.end_time, '%Y-%m-%d') AS end_time,
        c.status,
        c.tag,
        c.enroll_count
    FROM t_camp c
    LEFT JOIN t_camp_type ct ON c.type_id = ct.type_id
    <where>
        <if test="keyword != null and keyword != ''">
            AND c.name LIKE CONCAT('%', #{keyword}, '%')
        </if>
        <if test="status != null">
            AND c.status = #{status}
        </if>
        <if test="typeId != null">
            AND c.type_id = #{typeId}
        </if>
    </where>
    ORDER BY c.start_time DESC
    LIMIT #{offset}, #{limit}
</select>
```

**关键变更**:
- ✅ SELECT 子句中添加 `c.type_id`
- ✅ ResultMap 中添加 `typeId` 与 `type_id` 的映射关系
- ✅ 保持与 `t_camp_type` 表的左连接查询

## 数据库表结构对照

### t_camp 表
```sql
CREATE TABLE t_camp (
  camp_id INT PRIMARY KEY AUTO_INCREMENT,
  type_id INT NOT NULL,           -- ⭐ 外键，关联 t_camp_type.type_id
  term INT NOT NULL,
  name VARCHAR(100) NOT NULL,
  intro TEXT,
  start_time DATETIME NOT NULL,
  end_time DATETIME NOT NULL,
  status TINYINT DEFAULT 0,
  tag VARCHAR(50),
  enroll_count INT DEFAULT 0
);
```

### t_camp_type 表
```sql
CREATE TABLE t_camp_type (
  type_id INT PRIMARY KEY AUTO_INCREMENT,
  level_name VARCHAR(50) NOT NULL  -- 体系名称：诚意班、明理班等
);
```

## 注意事项

### ⚠️ **重要提醒**

1. **typeId 不能为空**
   - 前端编辑时必须使用 `typeId` 来回显体系类型下拉框
   - 如果缺失会导致无法精准匹配用户选择的体系

2. **字段命名一致性**
   - 数据库：`type_id`（下划线命名）
   - Java/JSON：`typeId`（驼峰命名）
   - 通过 MyBatis ResultMap 自动转换

3. **联表查询性能**
   - 使用 `LEFT JOIN` 确保即使 `type_id` 为 NULL 也能查询到营期数据
   - 建议为 `c.type_id` 添加索引优化查询性能
