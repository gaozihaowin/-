## 三、根因总结

### 3.1 问题模式

所有 22 个错误均属于**"接口契约不一致"**问题，总结为以下三种模式：

| 模式 | 说明 | 出现次数 |
|-----|------|--------|
| **模式 A：方法从未创建** | Service/Mapper 接口层完全缺少该方法声明 | 最多（17 个） |
| **模式 B：方法有 Java 接口，无 XML 实现** | 接口声明存在，但 XML 中没有对应 SQL | 3 个（HomeworkMapper） |
| **模式 C：方法有 XML SQL，但接口声明缺失** | XML 里有 `<select>` 标签，但 Java 接口没有声明方法 | 0 个 |

### 3.2 涉及的功能模块

| 模块 | 状态 | 影响 |
|-----|------|------|
| 营期分班管理（AdminClassController） | **完全不可用** | 7 个接口 404 |
| 升班检查与结营（PromotionServiceImpl） | **部分不可用** | 升班审核流程中断 |
| 管理员全息档案（SystemAdminServiceImpl） | **部分不可用** | assign/revoke 审计日志写入失败 |
| 证书颁发（CertificateService） | **接口缺失** | 结营无法自动发证 |

### 3.3 优先级建议

| 优先级 | 缺失项 | 理由 |
|-------|--------|------|
| P0 | `AdminDutyApplicationMapper.insertApplicationAudit` | assign/revoke 事务不完整，数据不一致 |

| P1 | `ClassService` 的 7 个方法 | 管理后台分班功能完全不可用 |

### 4.2 注意事项

1. **XML namespace 必须匹配**：新增的 Mapper 方法若有 XML 实现，`namespace` 必须严格指向对应的 Java 接口全类名（如 `com.daily.dailychineseculture.mapper.CampEnrollmentMapper`）
2. **事务一致性**：`SystemAdminServiceImpl.assign/revoke` 已标注 `@Transactional`，新增的 `insertApplicationAudit` 应在同一个事务中执行
3. **NULL 防御**：`selectByUserIdAndCampId` 等方法返回 null 时，调用方（如 `PromotionServiceImpl.checkPromotionEligibility`）已有判空处理，无需额外修改调用方代码
4. **批量操作的性能**：`selectEnrollmentsByCampId` 返回量较大时注意分页或流式处理
