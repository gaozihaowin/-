# 用户信息管理

## 功能名称
**用户个人资料查询与全量更新功能**

## 业务逻辑
该功能供用户在 App 端查看及编辑个人资料：
1. **获取个人概况**：在“我的”页面显示头像、昵称、学时统计（地区、职业、学时等）。
2. **获取详细资料**：进入编辑页面时获取全量字段信息（包括手机号、地区、生日、职业等）。
3. **全量更新资料**：一次性提交所有修改字段进行保存。
4. **修改头像与昵称**：提供独立接口仅修改基础信息。

## 技术实现

### 核心 Controller 实现
`AuthController` 提供个人概况、资料详情获取及全量更新的 RESTful 接口。

```java
// AuthController.java - 获取个人概况
@GetMapping("/user/info")
public Result<UserProfileDTO> getUserInfo(@RequestHeader("Authorization") String token) {
    Long userId = jwtUtils.getUserIdFromToken(token.replace("Bearer ", ""));
    UserProfileDTO userProfile = userService.getUserProfile(userId);
    return Result.success(userProfile);
}

// AuthController.java - 获取资料详情
@GetMapping("/user/detail")
public Result<UserDetailDTO> getUserDetail(@RequestHeader("Authorization") String token) {
    Long userId = jwtUtils.getUserIdFromToken(token.replace("Bearer ", ""));
    UserDetailDTO userDetail = userService.getUserDetail(userId);
    return Result.success(userDetail);
}

// AuthController.java - 全量更新资料
@PostMapping("/user/updateAll")
public Result<Void> updateUserAllInfo(
        @RequestHeader("Authorization") String token,
        @RequestBody UserUpdateAllRequest request) {
    Long userId = jwtUtils.getUserIdFromToken(token.replace("Bearer ", ""));
    boolean success = userService.updateUserAllInfo(userId, request);
    return success ? Result.successMsg("保存成功") : Result.error("保存失败");
}
```

### 核心 Service 逻辑
`UserService` 负责 DTO 与实体的转换、统计数据的计算以及数据库更新。

```java
// UserService.java - 获取概况并计算统计指标
public UserProfileDTO getUserProfile(Long userId) {
    User user = userMapper.selectById(userId);
    UserProfileDTO profileDTO = new UserProfileDTO();
    // 设置基本信息...
    profileDTO.setNickname(user.getNickname() != null ? user.getNickname() : user.getAccount());

    // 组装统计指标列表 (地区、职业、年数、学时)
    List<UserStatsItem> statsList = new ArrayList<>();
    statsList.add(new UserStatsItem("地区", user.getRegion()));
    statsList.add(new UserStatsItem("职业", user.getProfession()));
    
    // 计算学时：作业次数 * 2
    Integer homeworkCount = userMapper.countUserHomework(userId);
    int studyHours = (homeworkCount != null ? homeworkCount : 0) * 2;
    statsList.add(new UserStatsItem("学时", studyHours + "h"));

    profileDTO.setStatsList(statsList);
    return profileDTO;
}

// UserService.java - 全量更新逻辑
@Transactional(rollbackFor = Exception.class)
public boolean updateUserAllInfo(Long userId, UserUpdateAllRequest request) {
    User existingUser = userMapper.selectById(userId);
    // 映射字段
    if (request.getNickname() != null) existingUser.setNickname(request.getNickname());
    if (request.getAvatar() != null) existingUser.setAvatar(request.getAvatar());
    // 生日解析
    if (request.getBirthday() != null && !request.getBirthday().trim().isEmpty()) {
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd");
        existingUser.setBirthday(sdf.parse(request.getBirthday()));
    }
    // 密码处理：若不为空则更新，为空则保持原样
    if (request.getPassword() != null && !request.getPassword().trim().isEmpty()) {
        existingUser.setPassword(request.getPassword());
    }
    
    return userMapper.update(existingUser) > 0;
}
```

## 接口定义
- **获取个人概况**：`GET /user/info`
  - 返回：`UserProfileDTO` (包含 `statsList`)
- **获取资料详情**：`GET /user/detail`
  - 返回：`UserDetailDTO` (包含手机号、地区、性别等)
- **全量更新资料**：`POST /user/updateAll`
  - 参数：`UserUpdateAllRequest`
- **基础信息修改**：`POST /updateUserInfo`
  - 参数：`Map<String, String>` (仅限 `nickname`, `avatar`)

## 实现细节
- **学时统计算法**：系统不直接存储总学时，而是通过 `userMapper.countUserHomework(userId)` 实时统计用户提交成功的作业数量，每份作业按 2 小时折算。
- **注册年数计算**：基于 `user.createTime` 与当前系统时间的毫秒差，除以每年的毫秒数（365天）得出，向下取整。
- **密码安全策略**：在 `updateUserAllInfo` 中，若 `password` 字段为空字符串，则业务层不会修改数据库中的原密码，确保了编辑资料时不必每次输入密码。
- **数据回显脱敏**：`getUserDetail` 接口返回的 `UserDetailDTO` 中，`password` 字段始终设为空字符串，防止敏感信息泄露到前端。
