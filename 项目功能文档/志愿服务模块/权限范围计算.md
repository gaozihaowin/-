# 权限范围计算

**功能名称**：志愿服务权限范围计算

**业务逻辑**：
根据用户担任的志愿者职位，动态计算其管理权限覆盖的组织单元：
1. **职责任命识别**：确定用户在当前营期中担任的具体职位（如学班、检委、学组等）。
2. **管辖范围映射**：根据职位对应的级别，映射出具体的班级、大组或小组。
3. **动态展示控制**：前端基于返回的权限范围（Scopes）展示管理入口或筛选菜单。

**技术实现**：

### Controller 层
在 `VolunteerManageController` 中定义了获取管理范围的接口：

```java
    /**
     * 获取用户的所有管理范围
     * GET /volunteer/scopes
     */
    @GetMapping("/scopes")
    public ResponseResult<List<Map<String, Object>>> getManagementScopes(@RequestHeader("Authorization") String token) {
        try {
            Long userId = jwtUtils.getUserIdFromToken(token.replace("Bearer ", ""));
            List<Map<String, Object>> scopes = volunteerManageService.getManagementScopes(userId);
            return ResponseResult.success(scopes);
        } catch (Exception e) {
            return ResponseResult.error("获取管理范围失败：" + e.getMessage());
        }
    }
```

### Service 层
在 `VolunteerManageServiceImpl` 中封装调用逻辑：

```java
@Service
public class VolunteerManageServiceImpl implements VolunteerManageService {

    @Autowired
    private VolunteerManageMapper volunteerManageMapper;

    @Override
    public List<Map<String, Object>> getManagementScopes(Long userId) {
        // 直接从数据库查询真实的管理范围数据（包含层级关系）
        return volunteerManageMapper.getManagementScope(userId);
    }
}
```

### Mapper 层 (SQL)
在 `VolunteerManageMapper` 中通过 `COALESCE` 函数处理多层级组织间的嵌套关系：

```java
    @Select("SELECT DISTINCT da.assignment_id AS assignmentId, c.name AS campName, " +
            "COALESCE(cl.name, bg_class.name) AS className, " +
            "COALESCE(bg.name, sg_big.name) AS bigGroupName, " +
            "sg.name AS smallGroupName, da.duty_type AS dutyType " +
            "FROM t_duty_assignment da " +
            "LEFT JOIN t_duty_scope ds ON da.assignment_id = ds.assignment_id " +
            "LEFT JOIN t_camp c ON da.camp_id = c.camp_id " +
            "LEFT JOIN t_class cl ON ds.target_id = cl.class_id AND ds.target_type = 'class' " +
            "WHERE da.user_id = #{userId} AND da.volunteer_end_time IS NULL " +
            "AND c.end_time > NOW()")
    List<Map<String, Object>> getManagementScope(@Param("userId") Long userId);
```

**接口定义**：
- `GET /volunteer/scopes`：返回用户当前生效的所有管理权限范围。

**实现细节**：
1. **职责透传**：如果用户是“学班”（班级管理者），其 Scope 会包含 `classId`，后端业务逻辑据此判定其对该班级下所有子节点拥有权限。
2. **动态过滤**：SQL 层面显式过滤了 `volunteer_end_time IS NULL` 和营期结束时间，确保返回的权限具有实时有效性。
3. **多身份支持**：同一用户可以在不同营期担任不同职位，接口返回的是一个 Scope 列表，支持前端进行多营期、多职位的平滑切换。
4. **统一命名**：后端将不同职位的权限抽象为统一的 `targetType`（class/big_group/small_group），便于前端 UI 组件（如级联选择器）的通用化开发。
