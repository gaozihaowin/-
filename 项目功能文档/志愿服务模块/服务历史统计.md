# 服务历史统计

**功能名称**：志愿者服务历史统计

**业务逻辑**：
该功能向用户展示其在平台中担任志愿者的全历程，并提供汇总后的统计数据：
1. **志愿者历程**：详细展示每一项职责任命（Assignment），包括营期、职位名称、管辖范围及服务日期段。
2. **正在参与**：识别出当前用户正在担任的职务，并标记为“正在参与”。
3. **担当统计**：对用户参与过的营期总数、累计担当次数等进行汇总显示。
4. **退出担当**：支持用户或管理者发起退出当前职务的操作，记录服务结束时间。

**技术实现**：

### Controller 层
在 `VolunteerController` 中定义了志愿者记录查询接口：

```java
    /**
     * 获取用户志愿者历史记录
     * GET /user/volunteer-history
     */
    @GetMapping("/user/volunteer-history")
    public ResponseResult<VolunteerHistoryDTO> getVolunteerHistory(@RequestHeader("Authorization") String token) {
        Long userId = jwtUtils.getUserIdFromToken(token.replace("Bearer ", ""));
        VolunteerHistoryDTO result = userService.getVolunteerHistory(userId);
        return ResponseResult.success(result);
    }

    /**
     * 退出担当（更新志愿者服务结束时间）
     * POST /user/volunteer-quit
     */
    @PostMapping("/user/volunteer-quit")
    public ResponseResult<String> quitVolunteerDuty(@RequestHeader("Authorization") String token,
                                                    @RequestBody Map<String, Object> request) {
        Long userId = jwtUtils.getUserIdFromToken(token.replace("Bearer ", ""));
        Integer assignmentId = (Integer) request.get("assignmentId");
        boolean success = userService.quitVolunteerDuty(userId, assignmentId);
        return success ? ResponseResult.success("退出担当成功") : ResponseResult.error("操作失败");
    }
```

### Service 层
在 `UserService` 中实现了状态实时计算逻辑：

```java
    public VolunteerHistoryDTO getVolunteerHistory(Long userId) {
        List<Map<String, Object>> historyList = userMapper.getVolunteerHistory(userId);
        VolunteerHistoryDTO result = new VolunteerHistoryDTO();
        
        for (Map<String, Object> item : historyList) {
            String quitTime = (String) item.get("quitTime");
            Long rawCampEndTime = (Long) item.get("rawCampEndTime");
            
            // 核心逻辑：营期未结束且未主动退出，则为“正在参与”
            boolean isActive = (rawCampEndTime != null && System.currentTimeMillis() < rawCampEndTime * 1000) 
                               && "未设置".equals(quitTime);

            VolunteerHistoryItem hi = new VolunteerHistoryItem();
            hi.setAssignmentId((Integer)item.get("assignmentId"));
            hi.setResponsible((String)item.get("fullTargetName"));
            hi.setDuty((String)item.get("dutyName"));
            hi.setStatus(isActive ? "正在参与" : "已结束");
            result.getVolunteerHistory().add(hi);
        }
        return result;
    }
```

### Mapper 层 (SQL)
在 `UserMapper` 中通过复杂的 `CASE WHEN` 拼接权限范围名称：

```java
    @Select("SELECT da.assignment_id, c.name as campName, " +
            "CASE " +
            "    WHEN ds.target_type = 'class' THEN CONCAT(c.name, ' - ', cl.name) " +
            "    WHEN ds.target_type = 'big_group' THEN CONCAT(c.name, ' - ', bg.name) " +
            "    ELSE '未知职责范围' " +
            "END AS fullTargetName " +
            "FROM t_duty_assignment da " +
            "LEFT JOIN t_duty_scope ds ON da.assignment_id = ds.assignment_id " +
            "LEFT JOIN t_camp c ON da.camp_id = c.camp_id " +
            "WHERE da.user_id = #{userId}")
    List<Map<String, Object>> getVolunteerHistory(Long userId);
```

**接口定义**：
- `GET /user/volunteer-history`：返回志愿者历程列表。
- `GET /user/volunteer-stats`：返回参与营期及职位的统计汇总。
- `POST /user/volunteer-quit`：逻辑上结束某项职责任命。

**实现细节**：
1. **状态自动归档**：系统会对比当前时间与营期的 `end_time`，若营期已过，系统会自动将职位状态展示为“已结束”。
2. **软删除策略**：`quit` 操作通过更新 `volunteer_end_time` 字段实现，保留了历史服务记录用于年底勋章或证书颁发。
3. **多维统计**：`getVolunteerStats` 会分别按营期、班级、大组、小组四个维度聚合用户的服务历程。
4. **时间线呈现**：记录按 `volunteer_start_time` 倒序排列，确保用户首先看到最新的担当。
