# 统计分析

**功能名称**：作业统计分析

**业务逻辑**：
该功能为志愿者（管理角色）提供了对所辖学员作业完成情况的全局视野：
1. **实时统计**：按日统计当前范围内的作业总数、已完成、未完成、迟交等人数及百分比。
2. **多态名单**：提供已交名单、未交名单、迟交名单，方便点名及督促学习。
3. **数据钻取**：可通过关键词搜索特定学员在统计周期内的提交状态。

**技术实现**：

### Controller 层
在 `HomeworkController` 中定义了统计相关的核心接口：

```java
    /**
     * 获取作业统计数据
     * GET /homework/stats
     */
    @GetMapping("/homework/stats")
    public Result<Map<String, Object>> getHomeworkStatistics(@RequestHeader("Authorization") String token,
                                                             @RequestParam String type,
                                                             @RequestParam Integer id,
                                                             @RequestParam(value = "date", required = false) String date,
                                                             @RequestParam(value = "keyword", required = false) String keyword) {
        Long userId = jwtUtils.getUserIdFromToken(token.replace("Bearer ", ""));
        Map<String, Object> statistics = homeworkService.getHomeworkStatistics(userId, type, id, date, keyword);
        return Result.success(statistics);
    }

    /**
     * 获取作业状态名单
     * GET /homework/status/list/{type}/{id}
     */
    @GetMapping("/homework/status/list/{type}/{id}")
    public Result<Map<String, Object>> getHomeworkStatusList(@RequestHeader("Authorization") String token,
                                                             @PathVariable String type,
                                                             @PathVariable Integer id,
                                                             @RequestParam(value = "date", required = true) String date,
                                                             @RequestParam(value = "keyword", required = false) String keyword) {
        Long userId = jwtUtils.getUserIdFromToken(token.replace("Bearer ", ""));
        Map<String, Object> statusList = homeworkService.getHomeworkStatusList(userId, type, id, date, keyword);
        return Result.success(statusList);
    }
```

### Service 层
在 `HomeworkServiceImpl` 中执行聚合统计逻辑：

```java
    @Override
    public Map<String, Object> getHomeworkStatistics(Long userId, String type, Integer id, String date, String searchKeyword) {
        checkVolunteerAuth(userId, type, id); // 鉴权
        List<Long> studentIds = homeworkMapper.getStudentIdsByScope(type, id);
        
        // 1. 获取各类名单
        List<Map<String, Object>> submitted = homeworkMapper.getSubmittedHomeworkList(studentIds, date);
        List<Map<String, Object>> late = homeworkMapper.getLateHomeworkList(studentIds, date);
        List<Map<String, Object>> notSubmitted = homeworkMapper.getNotSubmittedHomeworkList(studentIds, date);

        // 2. 计算各项指标
        int total = studentIds.size();
        int completed = submitted.size() + late.size();
        
        Map<String, Object> stats = new HashMap<>();
        stats.put("totalCount", total);
        stats.put("completedCount", completed);
        stats.put("completionRate", total > 0 ? (completed * 100) / total : 0);
        stats.put("onTimeRate", completed > 0 ? (submitted.size() * 100) / completed : 0);
        return stats;
    }
```

### Mapper 层 (SQL)
在 `HomeworkMapper` 中通过 `NOT IN` 或集合对比实现“未交名单”的查询：

**接口定义**：
- `GET /homework/stats`：作业完成率及人数汇总。
- `GET /homework/status/list/{type}/{id}`：分类展示（已交、未交、迟交）的学员详细名单。

**实现细节**：
1. **计划关联性**：统计逻辑首先通过 `getPlanIdByDate` 确定目标日期是否有发布计划，若无计划则返回 `hasHomework: false`，避免空统计。
2. **多态数据聚合**：Service 层一次性获取多个分类名单，并在内存中完成百分比计算，减少了前端多次请求的压力。
3. **模糊检索支持**：支持在统计名单中通过 `keyword` 快速定位某个学员，实现精准催更。
