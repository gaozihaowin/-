# 任务完成与记录

## 功能名称
**学员每日学习任务点击与完成记录功能**

## 业务逻辑
该功能记录学员在 App 端的日常学习活动：
1. **任务打卡**：学员点击任务（如观看视频、阅读文档）后，系统自动记录为“已点击”或“已完成”。
2. **打卡状态展示**：在每日课表展示页面，显示每个任务的完成状态（待打卡、已完成）。
3. **进度统计**：基于任务完成记录，计算学员在该营期的整体活跃度和学习进度。

## 技术实现

### Controller 层
在 `AppCourseController` 中定义了提交任务打卡的接口：

```java
    /**
     * 完成任务打卡并返回最新进度
     * POST /courses/plan/{planId}/task/complete
     * 
     * @param planId 计划 ID
     * @param req 请求参数 (包含 taskId, campId)
     * @return 任务完成响应
     */
    @PostMapping("/plan/{planId}/task/complete")
   public Result<TaskCompleteRespDTO> completeTask(@PathVariable Integer planId, @RequestBody TaskCompleteReqDTO req, HttpServletRequest request) {
        Long currentUserId = (Long) request.getAttribute("userId");
        if (currentUserId == null) {
            throw new RuntimeException("用户未登录或 Token 失效，无法打卡");
        }
        TaskCompleteRespDTO resp = courseService.completeTask(planId, req, currentUserId);
        return Result.success(resp);
    }
```

### Service 层
在 `CourseServiceImpl` 中处理打卡持久化及进度实时计算：

```java
    @Override
    @Transactional(rollbackFor = Exception.class)
    public TaskCompleteRespDTO completeTask(Integer planId, TaskCompleteReqDTO req, Long currentUserId) {
        // 1. 记录打卡流水
        UserTaskRecord record = new UserTaskRecord();
        record.setUserId(currentUserId);
        record.setTaskId(req.getTaskId());
        record.setPlanId(planId);
        record.setCampId(req.getCampId());
        record.setCompleteTime(new Date());
        
        userTaskRecordMapper.insertIgnore(record); // 使用 INSERT IGNORE 防止重复
        
        // 2. 重新计算该营期的学习进度
        int totalTasks = planTaskMapper.countByCampId(req.getCampId());
        int completedTasks = userTaskRecordMapper.countCompletedByUserIdAndCampId(currentUserId, req.getCampId());
        
        double progress = totalTasks > 0 ? (double) completedTasks / totalTasks * 100 : 0;
        
        TaskCompleteRespDTO resp = new TaskCompleteRespDTO();
        resp.setProgress(progress);
        resp.setCompletedCount(completedTasks);
        return resp;
    }
```

### Mapper 层 (SQL)
在 `UserTaskRecordMapper.xml` 中通过 `INSERT IGNORE` 保证幂等性：

```xml
    <insert id="insertIgnore">
        INSERT IGNORE INTO t_user_task_record (user_id, task_id, plan_id, camp_id, complete_time)
        VALUES (#{userId}, #{taskId}, #{planId}, #{campId}, #{completeTime})
    </insert>
```


## 实现细节
- **幂等性设计**：通过数据库唯一索引（`userId`, `taskId`, `campId`）配合 `INSERT IGNORE` 指令，有效防止了用户因网络波动或重复点击导致的多次打卡记录。
- **实时进度反馈**：打卡成功后立即计算并返回该用户在该营期的最新进度百分比，实现了“即时反馈”的用户体验。
- **事务管理**：打卡操作使用 `@Transactional`，确保数据记录与进度计算逻辑的执行一致性。
- **无感记录**：该功能不要求用户输入内容，仅通过点击行为触发，极大降低了学员的使用成本。
