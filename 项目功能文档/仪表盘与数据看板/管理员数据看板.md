# 管理员数据看板

## 功能名称
**PC 后台管理仪表盘及活跃课程监控功能**

## 业务逻辑
该功能在 PC 管理端登录后的首页展示，供管理员宏观掌控教务情况：
1. **最近活跃课程展示**：实时展示最近有用户参与或新发布的营期列表。
2. **核心指标统计**：展示总学员数、进行中营期数、今日新增报名数等。
3. **快速导航入口**：提供营期管理、班级管理、作业评选等核心模块的快捷入口。

## 技术实现

### Controller 层
在 `AdminController` 中定义了获取最近活跃课程的接口：

```java
    /**
     * 获取最近活跃课程列表（仪表盘用）
     * GET /api/admin/dashboard/recent-camps
     * 
     * @return 最近活跃课程列表
     */
    @GetMapping("/dashboard/recent-camps")
    public Result<List<RecentCampDTO>> getRecentCamps() {
        try {
            List<RecentCampDTO> recentCamps = campService.getRecentCamps();
            return Result.success(recentCamps);
        } catch (Exception e) {
            e.printStackTrace();
            return Result.error("获取最新课程失败：" + e.getMessage());
        }
    }
```

### Service 层
在 `CampServiceImpl` 中处理最近活跃课程的逻辑，包含状态码到文本的转换：

```java
    @Override
    public List<RecentCampDTO> getRecentCamps() {
        // 查询最近活跃的 5 个营期
        List<Camp> camps = campMapper.selectRecentCamps();
        
        // 转换为 DTO 并处理状态文本
        List<RecentCampDTO> result = new ArrayList<>();
        SimpleDateFormat sdf = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
        
        for (Camp camp : camps) {
            RecentCampDTO dto = new RecentCampDTO();
            dto.setCampId(camp.getCampId());
            dto.setCampName(camp.getName());
            dto.setVisitCount(camp.getEnrollCount() != null ? camp.getEnrollCount() : 0);
            dto.setStatusCode(camp.getStatus() != null ? camp.getStatus() : 0);
            dto.setStatusText(getStatusText(camp.getStatus()));
            dto.setStartTime(camp.getStartTime() != null ? sdf.format(camp.getStartTime()) : "");
            
            result.add(dto);
        }
        
        return result;
    }

    private String getStatusText(Integer status) {
        if (status == null) return "待开课";
        switch (status) {
            case 0: return "待开课";
            case 1: return "进行中";
            case 2: return "已结束";
            default: return "未知状态";
        }
    }
```

### Mapper 层 (SQL)
在 `CampMapper.xml` 中实现查询（假设 `selectRecentCamps` 为按时间排序的查询）：

```xml
    <!-- 查询最近活跃课程 -->
    <select id="selectRecentCamps" resultMap="BaseResultMap">
        SELECT * FROM t_camp 
        ORDER BY start_time DESC 
        LIMIT 5
    </select>
```

## 接口定义
- **最近活跃课程接口**：`GET /api/admin/dashboard/recent-camps`
- **获取营期列表（大盘）**：`GET /api/admin/camps`

```java
@GetMapping("/dashboard/recent-camps")
public Result<List<RecentCampDTO>> getRecentCamps()
```

## 实现细节
- **状态转换**：后端维护了一套状态码枚举（0, 1, 2），在接口返回时自动转换为“待开课”、“进行中”等可读文本，减轻前端处理逻辑。
- **实时监控**：`visitCount` 字段映射自 `enroll_count`，反映了营期的实时热度。
- **日期格式化**：使用 `SimpleDateFormat` 统一输出标准日期字符串，确保 PC 端展示的一致性。
- **健壮性**：Controller 使用 try-catch 块包裹业务调用，并返回规范的 `Result` 对象，包含具体的报错信息。
