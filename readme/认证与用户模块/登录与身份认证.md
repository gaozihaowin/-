# 登录与身份认证

## 功能名称
**多端登录与身份切换功能**

## 业务逻辑
该功能支持多种登录方式：
1. **App 端账号密码登录**：用户输入账号和密码进行登录，若用户不存在则自动注册。
2. **App 端微信一键登录**：通过微信授权 code 获取 openid，实现静默登录。
3. **PC 管理端登录**：管理员专用登录入口，需校验管理权限。
4. **身份切换**：支持用户在不同端（App/PC）或同一端的不同角色（学员、志愿者）之间快速切换，切换后会颁发新的 JWT 令牌。

## 技术实现

### 核心 Controller 实现
`AuthController` 处理 App 端登录、退出及身份切换；`AdminController` 处理 PC 端管理登录。

```java
// AuthController.java - App 登录逻辑
@PostMapping("/login")
public Result<LoginResult> login(@RequestBody LoginRequest loginRequest) {
    try {
        // 步骤 1：参数校验
        if (loginRequest.getUsername() == null || loginRequest.getUsername().trim().isEmpty()) {
            return Result.error("请输入账号");
        }
        String username = loginRequest.getUsername().trim();
        String password = loginRequest.getPassword().trim();

        // 步骤 2：用户信息查询与比对
        boolean userExists = userService.checkUserExists(username);
        User user;
        if (userExists) {
            boolean passwordCorrect = userService.verifyPassword(username, password);
            if (!passwordCorrect) {
                return Result.build(401, "账号或密码错误", null);
            }
            user = userService.getUserByUsername(username);
        } else {
            // 用户不存在，自动注册
            user = userService.createUserWithReturn(username, password);
            if (user == null) {
                return Result.build(500, "用户注册失败", null);
            }
            return Result.build(201, "注册并登录成功", buildLoginResult(user, false));
        }

        // 步骤 3：判断信息完整度
        boolean isComplete = userService.isUserInfoComplete(user);
        return Result.success(buildLoginResult(user, isComplete));
    } catch (Exception e) {
        return Result.error("登录过程中发生错误: " + e.getMessage());
    }
}

// AdminController.java - PC 端登录
@PostMapping("/login")
public Result<AdminLoginResult> adminLogin(@RequestBody AdminLoginRequest request) {
    try {
        AdminLoginResult loginResult = adminAuthService.adminLogin(request);
        return Result.success(loginResult);
    } catch (RuntimeException e) {
        String errorMsg = e.getMessage();
        // 检查是否包含 403 标记（无权以该身份登录）
        if (errorMsg.contains(":403")) {
            return Result.build(403, errorMsg.replace(":403", ""), null);
        } else if ("账号或密码错误".equals(errorMsg)) {
            return Result.build(401, errorMsg, null);
        }
        return Result.build(400, errorMsg, null);
    }
}
```

### 核心 Service 逻辑
`UserAuthServiceImpl` 与 `AdminAuthServiceImpl` 负责具体的鉴权与身份切换逻辑。

```java
// UserAuthServiceImpl.java - 身份切换核心逻辑
@Override
public String executeIdentitySwitch(Long userId, SwitchIdentityRequest request, String clientType) {
    // 1. 校验 assignmentId 是否属于该用户
    Map<String, Object> assignment = dutyAssignmentMapper.selectById(request.getAssignmentId());
    if (assignment == null || !((Long) assignment.get("user_id")).equals(userId)) {
        throw new IllegalArgumentException("任命记录不存在或无权切换");
    }
    
    // 2. 获取新的 dutyType 和 campId
    String newDutyType = (String) assignment.get("duty_type");
    Integer campId = (Integer) assignment.get("camp_id");
    
    // 3. 构建 Token 载荷
    Map<String, Object> claims = new HashMap<>();
    claims.put("userId", userId);
    claims.put("dutyType", newDutyType);
    if (campId != null) claims.put("campId", campId);
    claims.put("clientType", clientType);
    
    // 4. 生成新 JWT Token
    return jwtUtils.generateToken(claims);
}
```

## 接口定义

### App 端接口
- **账号登录**：`POST /login`
  ```java
  public Result<LoginResult> login(@RequestBody LoginRequest loginRequest)
  ```
- **微信登录**：`POST /wxLogin`
  ```java
  public Result<Map<String, Object>> wxLogin(@RequestBody WxLoginRequest wxLoginRequest)
  ```
- **退出登录**：`POST /app/user/logout` (支持容错)
- **身份切换**：`POST /app/user/switch-identity`
  ```java
  public Result<Map<String, Object>> appSwitchIdentity(@RequestHeader("Authorization") String token, @RequestBody SwitchIdentityRequest request)
  ```

### PC 管理端接口
- **管理员登录**：`POST /api/admin/login`
  ```java
  public Result<AdminLoginResult> adminLogin(@RequestBody AdminLoginRequest request)
  ```

## 实现细节
- **自动注册**：在 `AuthController.login` 中，通过 `userService.checkUserExists` 判断。若为新用户，则调用 `createUserWithReturn`，并返回 HTTP 201 状态码，标记为“注册并登录成功”。
- **异常标识符**：`AdminAuthServiceImpl` 在抛出权限不足异常时会附带 `:403` 后缀。`AdminController` 捕获后解析该后缀并返回标准的 403 状态码。
- **JWT 扩展性**：身份切换通过 `jwtUtils.generateToken(claims)` 实现，支持在 Token 中动态携带 `dutyType`、`campId` 和 `clientType`，实现了无状态的权限控制。
- **信息完整性**：登录响应中包含 `isComplete` 字段，后端通过 `userService.isUserInfoComplete` 校验关键字段（如手机号、地区等）是否已填，引导用户补全资料。
