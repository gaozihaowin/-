# 作业评选与管理

**功能名称**：作业评选与管理

**业务逻辑**：
志愿者（具备管理角色的用户）可以在其管辖范围内（班级、大组或小组）执行以下操作：
1. **查看作业名单**：多维度筛选（全部/优秀）、日期筛选及关键字搜索管辖范围内的学员作业。
2. **查看作业详情**：查看特定作业的详细文本内容、提交时间及所属组织信息。
3. **评选优秀作业**：对表现突出的作业进行标记，标记后的作业会打上“优秀”标签，并可被单独筛选展示。

**技术实现**：

### Controller 层
在 `HomeworkController` 中定义了作业管理的核心接口：

```java
    /**
     * 获取作业名单
     * GET /homework/list/{type}/{id}/{status}
     */
    @GetMapping("/homework/list/{type}/{id}/{status}")
    public Result<HomeworkListDTO> getHomeworkList(@RequestHeader("Authorization") String token,
                                                   @PathVariable String type,
                                                   @PathVariable Integer id,
                                                   @PathVariable String status,
                                                   @RequestParam(value = "date", required = false) String date,
                                                   @RequestParam(value = "keyword", required = false) String keyword) {
        Long userId = jwtUtils.getUserIdFromToken(token.replace("Bearer ", ""));
        HomeworkListDTO homeworkList = homeworkService.getHomeworkList(userId, type, id, status, date, keyword);
        return Result.success(homeworkList);
    }

    /**
     * 标记优秀作业
     * POST /camp/homework/mark/{homeworkId}/{isExcellent}
     */
    @PostMapping("/camp/homework/mark/{homeworkId}/{isExcellent}")
    public Result<Void> markExcellentHomework(@RequestHeader("Authorization") String token,
                                              @PathVariable Integer homeworkId,
                                              @PathVariable Boolean isExcellent) {
        boolean success = homeworkService.markExcellentHomework(homeworkId, isExcellent);
        return success ? Result.successMsg("操作成功") : Result.error("作业不存在");
    }
```

### Service 层
在 `HomeworkServiceImpl` 中处理鉴权、数据过滤及组织信息动态拼接：

```java
    @Override
    public HomeworkListDTO getHomeworkList(Long userId, String type, Integer id, String status, String date, String searchKeyword) {
        checkVolunteerAuth(userId, type, id); // 严格鉴权
        List<Long> studentIds = homeworkMapper.getStudentIdsByScope(type, id);
        
        List<Map<String, Object>> rawList = homeworkMapper.getHomeworkList(studentIds, status, date, searchKeyword);
        List<HomeworkListDTO.HomeworkItem> items = new ArrayList<>();
        
        for (Map<String, Object> item : rawList) {
            // 动态拼接组织信息
            String organization = String.join("-", filterEmpty(
                (String)item.get("camp_name"), (String)item.get("class_name"), 
                (String)item.get("big_group_name"), (String)item.get("small_group_name")
            ));
            items.add(new HomeworkListDTO.HomeworkItem(
                (Integer)item.get("homeworkId"), (String)item.get("name"), 
                (Boolean)item.get("isExcellent"), formatTime(item.get("submit_time")), organization
            ));
        }
        return new HomeworkListDTO(items, items.size());
    }
```

### Mapper 层 (SQL)
使用 MyBatis 注解在 `HomeworkMapper` 中实现复杂的多表联查：

```java
    @Select("<script>" +
            "SELECT h.homework_id as homeworkId, u.user_id as userId, " +
            "       COALESCE(u.nickname, u.account) as name, h.is_excellent as isExcellent, " +
            "       h.submit_time, c.name as class_name, bg.name as big_group_name, " +
            "       sg.name as small_group_name, camp.name as camp_name " +
            "FROM t_homework h " +
            "LEFT JOIN t_user u ON h.user_id = u.user_id " +
            "LEFT JOIN t_camp_enrollment ce ON h.user_id = ce.user_id " +
            "LEFT JOIN t_class c ON ce.class_id = c.class_id " +
            "WHERE h.user_id IN " +
            "<foreach item='uid' collection='studentIds' open='(' separator=',' close=')'>#{uid}</foreach>" +
            "<if test='status == \"excellent\"'> AND h.is_excellent = true</if>" +
            "ORDER BY h.submit_time DESC" +
            "</script>")
    List<Map<String, Object>> getHomeworkList(@Param("studentIds") List<Long> studentIds, ...);
```

**接口定义**：
- `GET /homework/list/{type}/{id}/{status}`：获取作业列表。
- `POST /camp/homework/mark/{homeworkId}/{isExcellent}`：标记优秀作业。
- `GET /homework/detail/{homeworkId}`：查看作业详情。

**实现细节**：
1. **严格鉴权**：通过 `checkVolunteerAuth` 校验当前登录用户是否对请求的 `type` 和 `id` 拥有管辖权。
2. **多级组织感知**：SQL 关联了 `t_class`、`t_big_group`、`t_small_group`，使得志愿者可以一眼看到学员所属的具体小组。
3. **动态筛选**：支持按作业状态（全部/优秀）、提交日期及学员名称/学号进行实时检索。
